# VESTI Landing Reuse Map

This note documents how the VESTI landing prototype in this collection reuses the distilled lessons from the Midjourney landing study without copying Midjourney's brand surface.

Reference study artifacts:

- [Midjourney reference board](../midjourney/boards/2026-04-12T06-31-34-910Z/index.html)
- [Midjourney hero analysis](../midjourney/data/midjourney/analysis/2026-04-12T06-31-34-910Z/hero/hero-analysis.md)

## Principle Transfer

| Midjourney-derived principle | VESTI adaptation in this prototype | Why it fits VESTI |
| --- | --- | --- |
| Cinematic hero with a living text field | A custom `canvas` hero uses memory fragments, source tags, timestamps, semantic edges, and archive labels instead of Midjourney's numeric field. | VESTI is about captured thought and traceable recall, so motion should feel like memory structure, not ambient sci-fi wallpaper. |
| Wordmark plus atmospheric motion, treated as one composition | The `Vesti` wordmark, positioning line, metrics, and screenshot stack are all staged against one continuous motion field. | This makes the hero feel like a system surface, not a floating marketing block. |
| Screenshot-led storytelling | Real VESTI screenshots anchor the hero, workflow, capability grid, and trust section. | The README is feature rich. The fastest way to prove substance is to let the actual product UI carry the story. |
| Strong section rhythm | The page moves through hero, narrative, workflow, capabilities, architecture, trust, and CTA with clear tonal shifts. | Midjourney's landing page showed that section pacing is reusable even when the product is completely different. |
| Low-chrome navigation | Header chrome is restrained and secondary to the page narrative. | VESTI needs editorial focus more than a heavy marketing nav. |
| Typography contrast as hierarchy | Serif display, neutral sans body, and mono accents separate philosophy, product copy, and technical traces. | VESTI has both product philosophy and system architecture. The typography split makes those roles legible without adding extra UI noise. |
| Asymmetrical panel composition | Overlapping hero screenshots, offset trust visuals, and mixed card spans in the capability grid keep the page from collapsing into symmetric SaaS blocks. | VESTI's subject matter is richer than a standard feature checklist and benefits from a more editorial composition. |
| Motion with explicit restraint | Scroll reveals, light parallax, and a bounded hero animation all disable cleanly under reduced-motion settings. | The prototype proves motion can add atmosphere and hierarchy without making the page harder to read or less portable. |

## What Was Intentionally Not Reused

- Midjourney's dark monochrome skin was not copied. VESTI keeps a warm-paper base with a darker hero only where drama helps.
- Midjourney's numeric field was not copied. The hero canvas uses archive-specific language and graph-like traces instead.
- Midjourney's ambiguity-first copy style was not copied. VESTI needs sharper product explanation because its architecture and trust model are core to the pitch.

## Proof Standard

The prototype is considered a successful reuse proof if all three statements hold:

1. It feels more cinematic and more intentional than a generic product page.
2. It still reads unmistakably as VESTI rather than a Midjourney imitation.
3. A viewer can point to concrete, reusable landing mechanics beyond "it looks nice".
